# ADR-0001 B2B 受付モードと SNS モードの分離

## ステータス: Accepted (2026-07-10)

## 背景
- 現行 ja ウェルカムは「おっす！…やけん」で、士業・建設・製造の問い合わせ受付にはカジュアルすぎる（監査 §6）。
- Contact フォーム代用として使うには、構造化リードと機密入力禁止が必須。

## 決定

### モード
| モード | 用途 | トーン |
|---|---|---|
| **intake（本線）** | cor-jp.com お問い合わせ代用 | 敬語・簡潔・業務的 |
| **ambassador（副）** | SNS / イベント / デモ | 福岡弁・キャラ寄り可 |

- 本番 Contact 埋め込みは **intake 固定**。

### 初期文案（intake）
> こんにちは。Cor.株式会社のAI受付 Cloudiaです。5〜7問でご相談内容を整理し、担当者へ引き継ぎます。顧客名、個人情報、契約書本文などの機密情報は入力しないでください。まず、ご相談の目的を選んでください。

### 初期選択肢（intent）
- 機密データAI活用診断
- ローカルLLM / AI基盤PoC
- Grift Team Beta / Paid Trial
- Estimate Audit
- 取材・登壇・その他

### 会話フロー
1. 目的（intent）
2. 業種・役割・課題
3. データ種類と機密度（具体本文は入力させない）
4. 進捗段階
5. 希望時期・予算帯・意思決定者
6. 連絡先と個人情報同意
7. AI 要約 → 利用者修正 → 送信

### 保存
- 生会話全文の無期限保存はしない。要約と必要項目を主記録（詳細は corsweb PP / ADR-0005）。

## 理由
- B2B 購入判断者の第一印象とリード品質を優先する。
- SNS キャラ資産は破棄せずモード分離で残す。

## 影響
- `translations.ts` / プロンプト / UI のモード分岐。
- intent は corsweb 正本と同一キー。

## 代替案
- **全画面を福岡弁のまま**: 監査不合格のため却下。
